# Full-Stack Development Skill

## 适用场景
当你需要进行全栈开发时，包括前端、后端、数据库设计和 API 开发。

## 工作流程

### 1. 需求分析阶段
- 明确功能需求和业务逻辑
- 确定技术栈选型
- 设计数据模型和 API 接口
- 评估性能和安全要求

### 2. 架构设计
- 前后端分离架构
- RESTful API 或 GraphQL 设计
- 数据库 schema 设计
- 状态管理方案

### 3. 开发规范
- 遵循项目现有的代码规范
- 使用 TypeScript 进行类型安全开发
- 组件化开发，保持单一职责
- 编写清晰的注释和文档

### 4. 测试策略
- 单元测试覆盖核心逻辑
- 集成测试验证模块交互
- E2E 测试确保用户流程
- 性能测试和优化

### 5. 部署与运维
- CI/CD 流水线配置
- 环境变量管理
- 日志和监控
- 错误追踪

## 最佳实践

### 前端开发
- 使用 React/Vue 等现代框架
- 响应式设计，支持多端
- 性能优化（懒加载、代码分割）
- 无障碍访问（a11y）

### 后端开发
- API 版本控制
- 输入验证和 sanitization
- 认证授权（JWT/OAuth）
- 速率限制和防攻击

### 数据库
- 索引优化
- 查询性能分析
- 数据迁移脚本
- 备份策略

## 常用工具
- Next.js / Nuxt.js - 全栈框架
- Prisma / Drizzle - ORM
- TanStack Query - 数据获取
- Zod - 数据验证
- Docker - 容器化
