# Documentation Writing Skill

## 适用场景

当你需要编写项目文档、API 文档、技术规格书或用户手册时。

## 文档类型

### 1. README 文档

项目的门面，应该包含：

- 项目简介（一句话说明是什么）
- 核心功能列表
- 技术栈说明
- 快速开始指南
- 项目结构概览
- 贡献指南
- 许可证信息

### 2. API 文档

- 端点列表和描述
- 请求方法（GET/POST/PUT/DELETE）
- 请求参数（必填/选填、类型、示例）
- 响应格式和状态码
- 认证方式
- 错误处理
- 实际调用示例

### 3. 技术设计文档

- 背景和目标
- 系统架构图
- 数据流图
- 技术方案对比
- 实现细节
- 风险评估
- 时间规划

### 4. 用户手册

- 功能介绍
- 操作步骤（截图+说明）
- 常见问题 (FAQ)
- 故障排除
- 最佳实践

### 5. 开发规范文档

- 代码风格指南
- Git 工作流
- 命名规范
- 目录结构
- 测试要求
- Code Review 标准

## 写作原则

### 清晰简洁

- 使用简单直接的语言
- 避免行话和缩写（或提供解释）
- 一个段落只表达一个核心观点
- 使用主动语态

### 结构化

- 使用清晰的标题层级（H1 > H2 > H3）
- 列表优于长段落
- 重要信息加粗或使用引用块
- 提供目录导航（长文档）

### 可视化

- 使用图表说明复杂概念
- 代码示例要有语法高亮
- 截图标注关键区域
- 流程图展示操作步骤

### 可维护性

- 保持文档与代码同步
- 标注最后更新时间
- 提供反馈渠道
- 版本化文档（如有必要）

## Markdown 最佳实践

### 标题

```markdown
# 一级标题（文档标题，只用一次）
## 二级标题（主要章节）
### 三级标题（子章节）
```

### 代码块

````markdown
```typescript
// 指定语言，获得语法高亮
const greeting = "Hello, World!";
console.log(greeting);
```
````

### 表格

```markdown
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 用户姓名 |
| age | number | 否 | 年龄 |
```

### 引用和提示

```markdown
> **注意**: 这是一个重要的提示信息

💡 **提示**: 这里有个小技巧

⚠️ **警告**: 操作前请备份数据
```

## 文档模板

### 新功能文档模板

```markdown

# [功能名称]

## 概述

简要描述这个功能是什么，解决什么问题。

## 用户需求

- 用户故事 1
- 用户故事 2

## 技术方案

### 架构设计

[架构图或说明]

### 关键技术点

1. 技术点 1
2. 技术点 2

### 数据库变更

[Schema 变更说明]

## API 接口

### [端点名称]

- **URL**: `/api/xxx`
- **Method**: POST
- **Request**: {...}
- **Response**: {...}

## 测试计划

- [ ] 单元测试
- [ ] 集成测试
- [ ] E2E 测试

## 部署说明

[部署步骤和注意事项]

## 后续优化

[已知问题和未来改进方向]
```
