# Requirements Analysis Skill

## 适用场景
当你需要整理产品需求、分析业务逻辑或规划功能特性时。

## 需求收集方法

### 1. 用户访谈
- 目标用户是谁？
- 他们当前如何解决这个问题？
- 痛点在哪里？
- 期望的解决方案是什么？

### 2. 竞品分析
- 市场上有哪些类似产品？
- 他们的优缺点是什么？
- 我们的差异化优势？
- 可以借鉴的功能？

### 3. 数据分析
- 用户行为数据
- 转化率漏斗
- 留存率分析
- A/B 测试结果

## 需求分类

### MoSCoW 优先级
- **Must have** (必须有): 核心功能，没有就无法使用
- **Should have** (应该有): 重要但不是必须，影响用户体验
- **Could have** (可以有): 锦上添花，有时间再做
- **Won't have** (不会有): 本次不做，未来考虑

### Kano 模型
- **基本型需求**: 必须具备，不满足会非常不满意
- **期望型需求**: 越多越好，线性关系
- **兴奋型需求**: 超出预期，带来惊喜
- **无差异需求**: 有无都不影响满意度
- **反向需求**: 有了反而不满意

## 需求文档结构

### 1. 项目背景
```markdown
## 背景
[为什么要做这个功能/项目]

## 目标
[希望达成什么业务目标]

## 成功指标
- 指标 1: [具体数值]
- 指标 2: [具体数值]
```

### 2. 用户故事
```markdown
作为 [用户角色]
我想要 [执行某个操作]
以便于 [达成某个目标]

验收标准:
- [ ] 标准 1
- [ ] 标准 2
- [ ] 标准 3
```

### 3. 功能需求
```markdown
## 功能列表

### 功能 1: [名称]
**描述**: [详细说明]
**优先级**: P0/P1/P2
**依赖**: [依赖的其他功能]

**用户流程**:
1. 步骤 1
2. 步骤 2
3. 步骤 3

**边界情况**:
- 情况 1: [处理方式]
- 情况 2: [处理方式]
```

### 4. 非功能需求
```markdown
## 性能要求
- 页面加载时间 < 2s
- API 响应时间 < 500ms
- 支持并发用户数: 1000+

## 安全要求
- 数据加密传输
- 敏感信息脱敏
- 防 SQL 注入/XSS

## 兼容性
- 浏览器: Chrome, Firefox, Safari, Edge (最新 2 个版本)
- 移动端: iOS 13+, Android 8+
- 屏幕尺寸: 320px - 2560px
```

### 5. 技术方案
```markdown
## 技术选型
- 前端框架: [选择及原因]
- 后端技术: [选择及原因]
- 数据库: [选择及原因]

## 架构设计
[架构图和说明]

## 接口设计
[API 接口定义]

## 数据模型
[数据库 Schema]
```

### 6. 项目计划
```markdown
## 里程碑
- Phase 1: [日期] - [交付内容]
- Phase 2: [日期] - [交付内容]
- Phase 3: [日期] - [交付内容]

## 资源需求
- 开发人员: X 人
- 设计人员: X 人
- 测试人员: X 人

## 风险评估
| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 风险 1 | 高 | 高 | 应对方案 |
| 风险 2 | 中 | 中 | 应对方案 |
```

## 需求验证清单

### 完整性检查
- [ ] 所有用户场景都覆盖了吗？
- [ ] 边界情况都考虑了吗？
- [ ] 错误处理明确了吗？
- [ ] 依赖关系清晰吗？

### 可行性检查
- [ ] 技术可实现吗？
- [ ] 时间和资源充足吗？
- [ ] 符合业务目标吗？
- [ ] ROI 合理吗？

### 一致性检查
- [ ] 与现有系统兼容吗？
- [ ] 与其他需求冲突吗？
- [ ] 符合设计规范吗？
- [ ] 遵循技术标准吗？

## 需求变更管理

### 变更流程
1. 提出变更请求
2. 评估影响（时间、成本、风险）
3. 相关方审批
4. 更新文档
5. 通知团队
6. 调整计划

### 变更记录模板
```markdown
## 变更记录

| 日期 | 版本 | 变更内容 | 提出人 | 审批人 |
|------|------|----------|--------|--------|
| 2024-01-01 | v1.1 | 新增 XX 功能 | 张三 | 李四 |
```

## 工具推荐
- Jira / Trello - 需求管理
- Miro / FigJam - 脑图和流程图
- Notion / Confluence - 文档协作
- UserStory Map - 用户故事地图
- Draw.io - 流程图和架构图
